# Fluid Words — Agent 向けガイド

指で宇宙に言葉を書く、iPhone 向けの癒し系 Web アプリ。  
Vanilla HTML / CSS / JS（バンドラ・フレームワークなし）。公開先: GitHub Pages。

## 前提

- **第一ターゲットは iPhone Safari**（タッチ、視覚的ビューポート、ホーム画面追加、ハプティクス）
- 既存のフラット構成を崩さない。React / Vite / TypeScript / npm 依存は、依頼がない限り入れない
- UI 文言は **日本語メイン**、英語は補助。コメントも既存に合わせて日本語でよい
- 見た目・体験は「派手なゲーム UI」ではなく、静かな宇宙・やわらかいフィードバック

## 構成

```
index.html / styles.css / server.js
js/main.js      … 入力・演出オーケストレーション
js/fluid.js     … WebGL2 流体
js/ink.js       … 筆跡 + OCR
js/positive.js  … ポジティブ語
js/meteors.js / js/haptics.js
```

- 役割ごとの分割を保つ。大きな横断変更は `main.js` から辿る
- キャッシュ対策: `index.html` の `?v=` を触ったら版を上げる
- ローカル確認: `node server.js` → ポート **8090**

## 実装ルール

- 変更は最小差分。無関係なリファクタやファイル分割をしない
- タッチとマウスの両方を壊さない。`preventDefault` / スクロールロックは既存パターンに合わせる
- WebGL・OCR・ハプティクスは失敗してもアプリ全体が死なないようにする（既存のガードを踏襲）
- 設定値やスライダー端点はマジックナンバーを散らさず、既存の定数・STOPS 配列に足す
- やり取りは日本語。確認が必要なものを除き、区切りごとに commit して GitHub へ push する

## やってほしくないこと

- UI をカードだらけ・ダッシュボード風・紫グラデ定番に作り直す
- 依存追加やビルドパイプラインの導入（依頼なし）
- README / ドキュメントの勝手な大幅更新
